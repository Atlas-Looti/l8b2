# Types

LootiScript is a dynamically typed language. This means variables do not have types; only values have types. A variable can hold a value of any type at any time.

## Basic Types

LootiScript supports the following basic types:

| Type | Description | Example |
|------|-------------|---------|
| **Number** | Floating-point numbers (doubles) | `10`, `3.14`, `-5` |
| **String** | Text data | `"Hello"`, `'World'` |
| **List** | Ordered sequence of values (Array) | `{1, 2, 3}` |
| **Object** | Key-value pairs | `object x=1 end` |
| **Function** | Executable code block | `x => x * 2` |
| **Null** | Represents intentional absence of value | `null` |

## Type Checking

You can check the type of a value using the `type()` function or the `.type` property.

```lua
local x = 10
print(type(x))       // "number"

x = "Hello"
print(x.type)        // "string"

x = {1, 2}
print(type(x))       // "list"
```

## Type Details

### Numbers
All numbers in LootiScript are floating-point numbers. There is no distinction between integers and floats.

```lua
local a = 10
local b = 3.14
local c = 0xFF  // Hexadecimal
```

### Strings
Strings can be enclosed in single quotes `'` or double quotes `"`.

```lua
local s1 = "Double quotes"
local s2 = 'Single quotes'
```

#### String Concatenation
Use the `..` operator to concatenate strings:

```lua
local greeting = "Hello" .. " " .. "World"  // "Hello World"
local name = "Alice"
local msg = "Hello, " .. name  // "Hello, Alice"
```

#### Template Strings
Use backticks `` ` `` for string interpolation. Template strings allow you to embed expressions directly in the string.

```lua
local name = "Alice"
local age = 30
local msg = `Hello, ${name}! You are ${age} years old.`
// Result: "Hello, Alice! You are 30 years old."

// You can use any expression inside ${}
local x = 10
local y = 20
local result = `Sum: ${x + y}`  // "Sum: 30"
```

#### Multi-line Strings
Use triple quotes `"""` for strings that span multiple lines.

```lua
local text = """
This is a
multi-line
string.
"""
```

### Lists (Arrays)
Lists are ordered collections of values. They are **0-indexed**, meaning the first element is at index 0.

```lua
local items = {10, 20, 30}
print(items[0])  // 10 (first element)
print(items[1])  // 20 (second element)
print(items[2])  // 30 (third element)
```

**Note:** LootiScript uses 0-based indexing, unlike Lua which uses 1-based indexing. The first element is always at index 0.

### Objects
Objects are collections of key-value pairs.

```lua
local player = object
  name = "Hero",
  health = 100
end

print(player.name)    // "Hero"
print(player["name"]) // "Hero"
```

### Null vs 0
- **Uninitialized variables** are `0`.
- **`null`** is a specific value you can assign to indicate "nothing".

```lua
local x
print(x)      // 0

local y = null
print(y)      // null
```
